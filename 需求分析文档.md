# 校园活动发布平台需求分析文档（含User Story与优先级）

## 一、项目概述
本项目为**校园活动发布平台**，采用前后端分离架构（前端 Vue + 后端 Spring Boot 生态），旨在为校园用户（学生、教师、社团负责人、管理员）提供活动发布、查询、管理及互动的一体化解决方案。平台支持活动全生命周期管理，结合 AI 助手提升操作效率，通过消息队列实现状态自动化，满足校园场景下多样化的活动组织需求。

## 二、核心功能需求

### （一）用户模块
1. **身份认证**
   - 支持多角色登录（学生、教师、社团负责人、管理员），账号密码校验逻辑
   - 注册功能：密码采用 BCrypt 哈希加密存储，保障数据安全
   - 响应式登录 / 注册页面：支持鼠标交互动画（眼部、手部跟随鼠标移动），提升用户体验
   - 账号状态管理：包含正常、禁用、未激活三种状态，支持逻辑删除

2. **用户信息管理**
   - 存储用户基础信息（真实姓名、性别、联系方式、所属学校 / 学院 / 专业 / 年级）
   - 支持头像上传与展示（关联头像 URL 字段）
   - 记录用户最后登录时间与 IP 地址

### （二）活动模块
1. **活动基础操作（CRUD）**
   - 活动创建：支持填写活动名称、描述、起止时间、创建人等核心信息，上传封面图片
   - 活动编辑：允许修改活动基础信息，状态自动同步更新时间
   - 活动删除：支持逻辑删除与物理删除（根据权限控制）
   - 活动查询：支持按标题、内容、发起人模糊搜索，按时间范围、状态筛选

2. **活动特色功能**
   - 热门活动管理：新增`hot_status`字段，支持热门活动标记与筛选
   - 活动状态自动化：通过 RabbitMQ 实现活动状态自动切换（未开始→进行中→已结束）
   - 图片上传：创建活动时支持调用`POST/api/updown/img`接口上传图片，存储路径为`src/main/resources/static/img`

3. **活动展示**
   - 主页面公告展示功能
   - 热门活动页面专题展示
   - 活动二级页面与创建活动二级页面的可视化呈现（支持样式优化扩展）

### （三）通知模块
1. **系统通知管理**
   - 通知 CRUD 接口：支持新增、按 ID 删除 / 修改 / 查询、获取所有公告
   - 通知接收范围控制：支持所有人、普通用户、医生三类接收对象
   - 通知状态管理：区分未读、已读、已删除三种状态

### （四）AI 助手模块
1. **交互功能**
   - 导航栏 AI 助手按钮：点击打开可自由拖动的对话框
   - 自然语言交互：支持口语化对话输入
   - 接口适配：通过`POST/api/chat`接口实现 AI 功能对接

2. **核心能力**
   - 指令转译：利用大模型注意力机制将口语化对话转译为固定指令
   - 活动操作自动化：支持指令化创建活动、查询活动
   - 双业务逻辑板块：`ModelEngineService`（模型引擎）与`SmartAssistantService`（智能助手）协同工作

### （五）系统架构与依赖
1. **后端技术栈**
   - 核心框架：Spring Boot
   - 数据访问：MyBatis（数据库操作）
   - 缓存：Redis（用于 Token 存储，待完善）
   - 消息队列：RabbitMQ（活动状态自动切换）
   - 接口文档：Swagger（接口统一管理，本地访问地址：http://localhost:8080/swagger-ui/index.html）

2. **数据库设计**
   - 核心表：`sys_user`（用户表）、`sys_activity`（活动表）、`sys_notice`（通知表）
   - 字符集：utf8mb4（支持全量字符存储）
   - 存储引擎：InnoDB（支持事务与索引优化）
   - 索引优化：针对常用查询字段（时间、状态、用户类型等）建立索引

## 三、非功能需求

### （一）性能需求
1. 接口响应速度：基础查询接口响应时间≤300ms，图片上传接口响应时间≤2s
2. 并发支持：支持至少 100 人同时在线操作，活动创建 / 查询接口支持 50QPS
3. 数据存储：支持万级活动数据存储，图片存储支持单文件≤10MB

### （二）可用性需求
1. 响应式布局：支持 PC 端、平板等多终端访问
2. 容错性：接口支持异常场景处理（参数错误、数据重复、并发请求等）
3. 稳定性：消息队列经过测试，确保活动状态自动切换无遗漏
4. 可维护性：前后端分离架构，代码解耦，支持后续功能扩展

### （三）安全需求
1. 数据安全：用户密码加密存储，敏感信息（手机号、邮箱）脱敏展示
2. 权限控制：基于用户角色的访问控制（如管理员与普通用户操作权限区分）
3. 接口安全：支持 Token 验证（Redis 缓存待完善），防止非法访问

### （四）用户体验需求
1. 交互体验：导航栏支持折叠功能，操作流程简洁直观
2. 视觉体验：登录页面添加动态交互效果，活动页面样式优化（待完善）
3. 反馈机制：操作结果实时反馈，错误提示清晰明确

## 五、User Story 与优先级

### 优先级说明
- **高（P0）**：核心功能，影响平台基础可用性，必须优先实现
- **中（P1）**：重要功能，提升用户体验或扩展核心场景，核心功能稳定后实现
- **低（P2）**：优化类、拓展类功能，可在迭代中逐步完善

## 六、待完善需求与风险点

### （一）待完善功能
1. Redis 缓存集成：完善 Token 存储、读取、过期逻辑，移除 try catch 包裹
2. Swagger 文档访问：修复本地访问 URL 拼写错误（当前报错“URL拼写可能存在错误，请检查”），支持非本机环境查看
3. 异常场景测试：补充接口异常场景（参数错误、数据重复、并发请求等）的测试与处理
4. 样式优化：导航栏折叠后样式、活动二级页面样式、创建活动页面样式完善

### （二）潜在风险
1. 接口兼容性：前后端联调需确保 axios 请求与后端接口适配
2. 大模型稳定性：AI 助手的指令转译准确率需持续优化
3. 图片存储：大量图片上传可能导致存储压力，需考虑分布式存储扩展
4. 消息队列可靠性：需持续测试 RabbitMQ 在高并发场景下的消息投递成功率
5. Swagger 访问风险：URL 拼写错误可能导致接口文档无法访问，影响前后端协作效率
